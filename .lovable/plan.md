

# План: Сдвинуть левую страницу вправо к орнаменту (без изменения ширины)

## Изменение в `src/components/FinalBook.tsx` (строка 177, левая страница)

```
left:  "17.5%"  →  "20%"
width: "27.5%"  —  БЕЗ ИЗМЕНЕНИЙ
top, height, padding — БЕЗ ИЗМЕНЕНИЙ
```

Сдвиг на +2.5% вправо смещает начало текста к внутреннему краю левого орнамента (зеркально правой странице, у которой `left: 54%`). Ширина `27.5%` сохраняется по требованию — правый край блока станет 47.5% (раньше был 45%), что всё ещё в пределах центрального разворота и не задевает корешок.

## Что НЕ затрагивается
- `width` левой страницы (остаётся 27.5%)
- Правая страница (`left: 54%`, `width: 26%`)
- Стиль текста (`textAlign`, шрифт, lineHeight)
- Логика, данные, пагинация, реакции
- Вертикальные границы (top, height)

## Результат
- Текст левой страницы начинается строго от внутреннего края орнамента
- Все строки идут от одной вертикали
- Текст не заходит на орнамент
- Визуально левая страница совпадает с правой по позиции старта строк
