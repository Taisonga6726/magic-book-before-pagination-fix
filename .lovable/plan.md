

# План: Строгие границы текста по орнаменту в FinalBook

## Проблема (по скрину)
- Запись №13 (CI/CD pipeline) частично заходит на верхний орнамент
- Записи 14, 16, 17 выходят влево за линию орнамента
- Запись №18 (Ревью пройдено) и её реакции выходят вниз за нижний орнамент

Причина: контейнер слишком высокий (`top: 18%`, `height: 60%` = до 78%) и сдвинут слишком влево (`left: 17%`). Нижний орнамент начинается раньше 78%, верхний орнамент шире чем 18% сверху, а внутренний край левого орнамента правее `17%`.

## Изменение в `src/components/FinalBook.tsx` (строки 173–203)

### Левая страница (строки 176–179)
```
left:    "17%"  →  "19%"   (сдвиг вправо — текст не пересекает орнамент слева)
top:     "18%"  →  "21%"   (опускаем — освобождаем верхний орнамент)
width:   "29%"  →  "26%"   (правая граница 45%, как было)
height:  "60%"  →  "54%"   (нижняя граница 75% — выше нижнего орнамента)
padding: "12px 2px 40px 4px" → "8px 4px 24px 4px"
```

### Правая страница (строки 191–194) — синхронно по вертикали
Чтобы страницы были симметричны по высоте (вертикальные границы орнаментов одинаковы слева и справа):
```
top:     "18%"  →  "21%"
height:  "60%"  →  "54%"
padding: "12px 2px 40px 4px" → "8px 4px 24px 4px"
left, width — БЕЗ ИЗМЕНЕНИЙ ("54%", "26%")
```

### Пагинация (строки 43–45)
Логика измерения уже использует `container.clientHeight` — она автоматически подхватит новую высоту, поэтому записи будут переноситься на новую страницу при переполнении. Менять код не нужно.

Чтобы дать измерителю запас под номер страницы (находится `bottom: 4px` внутри контейнера, ~20px высоты), вычесть 24px из `availableHeight`:
```
const availableHeight = container.clientHeight - 24;  // запас под номер страницы
```
Это гарантирует, что последняя запись не наложится на « — 3 — ».

## Что НЕ затрагивается
- Данные, `entries`, `localStorage`
- `renderEntry`, реакции, шрифты, justify-выравнивание
- MagicBook
- Логика flip / навигации / звуков

## Результат
- Текст начинается строго от внутреннего края левого орнамента
- Верхняя запись не касается верхнего орнамента
- Нижняя запись (с реакциями) заканчивается выше нижнего орнамента
- Если запись не помещается — переносится на следующую страницу
- Левая и правая страницы симметричны по вертикали
