

## Диагноз

На скрине из Word: hanging indent. Номер «1.», «2.», «3.» в узкой колонке слева, текст (слово + описание) — в широкой колонке справа, **вторая строка описания идёт под началом текста, не под номером**.

Сейчас у нас на левой странице номер и слово в одной inline-строке (`{globalIdx + 1}. {renderInkWord(entry.word)}`), а описание — отдельным `<div>` во всю ширину контейнера. Из-за этого описание начинается от левого края (под номером), а не выровнено по слову. Это и есть «лесенка наоборот».

## Решение

В `src/components/FinalBook.tsx`, ветка `side === "left"`:

1. Двухколоночный flex с hanging indent:
   - левая колонка: номер «N.», `min-width: 1.6em`, `flex-shrink: 0`;
   - правая колонка (`flex-1`): слово, описание, реакции — всё внутри.
2. Между колонками естественный отступ ~`0.3em` (через `padding-right` на колонке номера) — никакого «лишнего пробела».
3. Описание перетекает под слово, а не под номер — точно как в Word.
4. Правую страницу (`side === "right"`) не трогаю.
5. Синхронизирую измерительный блок пагинации (строки ~58-66) под новые `min-width: 1.6em` + `padding-right: 0.3em`, чтобы переносы считались верно.

## Изменение (только левая ветка `renderEntry`)

```tsx
if (side === "left") {
  return (
    <div key={globalIdx} className="flex mb-0 w-full items-start">
      <div
        className="text-xl leading-tight font-bold"
        style={{
          color: "#1a1440",
          fontFamily: "'Cormorant Garamond', serif",
          fontStyle: "italic",
          lineHeight: "1.15",
          minWidth: "1.6em",
          flexShrink: 0,
          paddingRight: "0.3em",
          textAlign: "left",
        }}
      >
        {globalIdx + 1}.
      </div>
      <div className="flex-1 flex flex-col" style={{ minWidth: 0 }}>
        <div /* слово, как сейчас */>{renderInkWord(entry.word)}</div>
        {entry.description && (
          <div /* описание, как сейчас, но без своего left-padding */>
            — {entry.description.replace(/^[—–\-]\s*/, "")}
          </div>
        )}
        <div /* реакции, как сейчас */>...</div>
      </div>
    </div>
  );
}
```

## Что НЕ трогаю
- Правую страницу (`side === "right"`) — ноль изменений.
- Координаты страниц (18%/31%, 54%/26%), `top`, `height`.
- Шрифты, цвета, ink-эффект, нумерацию страниц, реакции.

## Результат
- Номера «1.», «2.», «3.» / «11.», «12.», «13.» в узкой колонке у орнамента, по одной вертикали.
- Между номером и словом — обычный одиночный пробел.
- Описание начинается под словом (hanging indent), а не под номером — как в Word.
