

# План: Визуальная проверка левой страницы FinalBook

## Что проверю
Открою превью в браузере на текущем разрешении (1311×861), перейду к экрану FinalBook (где видны записи на двух страницах), сделаю скриншот и оценю:

1. Левый край текста — совпадает ли с внутренним краем левого орнамента (без зазора, без наезда).
2. Правый край записей — не выходит ли за границы (не упирается в корешок, не перечёркивается орнаментом).
3. Все строки начинаются от одной вертикали.

## Шаги
1. `navigate_to_sandbox` на `/` с viewport 1311×861.
2. `observe` — найти, как попасть в режим FinalBook (нужна заполненная книга или прямой переход).
3. Если нужно — заполнить несколько записей или использовать существующее состояние из localStorage.
4. `screenshot` — зафиксировать левую страницу.
5. Если левый край всё ещё с зазором или правый вылезает — предложить корректировку `left` (например, 16% или 18%) и/или уменьшение `width`.

## Что доложу
- Скриншот + вывод: «выровнено по орнаменту / есть зазор слева / вылезает справа».
- Если есть проблема — конкретное предложение правки (новое значение `left`/`width` в `FinalBook.tsx`, строка 177).
