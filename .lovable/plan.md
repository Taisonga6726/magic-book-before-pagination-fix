

## Понял по референсу

Структура из Word на скрине:
```
1.  ВайБКОДчанин
    — это человек, который проходит курс в школе...
    
2.  КУРСОРчик
    - друг ВайБКОДчанина, который помогает...
```

Hanging indent: номер слева в узкой колонке, **слово справа от номера в той же строке**, описание под словом с тем же отступом, что и слово (не под номером). Между записями — пустая строка для дыхания. И так на **обеих** страницах.

## Что сейчас не так

1. **Левая страница**: между «N.» и словом зазор `min-width: 1.6em + padding-right: 0.3em` — это ~2 символа, выглядит как лишний пробел. Должно быть `min-width: 1.4em`, без `padding-right` — просто естественный пробел внутри flex.
2. **Правая страница**: совсем другая разметка (`width: 2.4em` колонка номера) — большой зазор + описание уезжает под номер (нет hanging indent). Нужно сделать **точно как левую**.
3. **Мало записей на странице (3–4)**: координаты `top: 21%, height: 54%` оставляют много пустого места снизу до орнамента. По скрину пользователя контент должен заполнять всё пространство до нижнего орнамента. Увеличиваю `height` до ~62% (или опускаю `top` до ~19%, расширяю `height` до 60%) — без захода на орнамент.
4. **Между записями нет дыхания** — `mb-0` слипает их. На референсе между «1.» и «2.» — пустая строка. Добавлю `marginBottom: 0.6em` между записями (и синхронизирую в `measure`).

## Правки в `src/components/FinalBook.tsx`

### 1. Унифицировать `renderEntry` для обеих сторон (одна разметка)

Удалить разделение `if (side === "left")` — обе стороны рендерят одинаково:

```tsx
const renderEntry = (entry: Entry, globalIdx: number) => (
  <div key={globalIdx} className="flex w-full items-start" style={{ marginBottom: "0.6em" }}>
    <div style={{
      color: "#1a1440",
      fontFamily: "'Cormorant Garamond', serif",
      fontStyle: "italic",
      fontWeight: 700,
      fontSize: "1.25rem",
      lineHeight: "1.15",
      minWidth: "1.4em",
      flexShrink: 0,
      textAlign: "left",
    }}>
      {globalIdx + 1}.
    </div>
    <div className="flex-1 flex flex-col" style={{ minWidth: 0 }}>
      <div style={{ /* слово: italic bold 1.25rem, lineHeight 1.15, color #1a1440 */ }}>
        {renderInkWord(entry.word)}
      </div>
      {entry.description && (
        <div style={{ /* описание: 1rem, lineHeight 1.15, color #1a1030 */ }}>
          — {entry.description.replace(/^[—–\-]\s*/, "")}
        </div>
      )}
      <div className="flex gap-2 text-[13px] justify-end">
        {/* реакции */}
      </div>
    </div>
  </div>
);
```

Изменения по сравнению с текущим:
- `minWidth: 1.4em` (было 1.6em), убрать `paddingRight: 0.3em` — естественный одиночный пробел.
- `marginBottom: 0.6em` между записями — дыхание как в Word.
- Правая страница использует ту же функцию — никакой особой разметки.

### 2. Заполнить страницу до орнамента

Координаты контейнеров:
- было: `top: "21%", height: "54%"`
- станет: `top: "19%", height: "60%"`

Это поднимет верх ближе к верхнему орнаменту и опустит низ ближе к нижнему — без наезда. Применить к **обоим** контейнерам (левому и правому).

### 3. Синхронизировать `measure` для пагинации

Строки 58–66 переписать ровно под новую разметку (`min-width: 1.4em`, `marginBottom: 0.6em`, без `padding-right`). Иначе пагинация снова будет считать неверно и недозаполнять страницу.

### 4. Сигнатура вызова

Убрать аргумент `side` из вызовов в строках 241 и 256: `renderEntry(entry, getGlobalIndex(entry))`.

## Что НЕ трогаю
- Орнамент, фон, координаты по горизонтали (18%/31%, 54%/26%).
- Шрифты, цвета, ink-эффект, нумерация страниц, перелистывание, реакции (только их позиционирование внутри правой колонки — `justify-end`, как сейчас).

## Результат
- Обе страницы — единый стиль hanging indent, как в Word.
- Между «N.» и словом обычный одиночный пробел.
- Описание под словом, не под номером.
- Между записями — пустая строка (дыхание).
- Контент заполняет всё пространство до орнамента, ~6–8 записей на страницу вместо 3–4.
