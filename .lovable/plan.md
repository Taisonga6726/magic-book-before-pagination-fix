

## Цель
Убрать автоматическую очистку словаря при первом запуске. Слова уже внесены — пользователь должен видеть их и продолжать список, а не получать пустую книгу.

## Анализ
Согласно memory `tech/session-management`, при первой загрузке проверяется флаг `magic-book-initialized` в `localStorage`. Если его нет — словарь (`entries`) принудительно очищается, и флаг ставится. Это поведение нужно отключить, чтобы существующие записи в `localStorage` сохранялись.

## Что проверить
- `src/pages/Index.tsx` — найти блок, где читается флаг `magic-book-initialized` и вызывается очистка `entries` / `localStorage.removeItem(...)`.

## План правки

**Файл: `src/pages/Index.tsx`**
- Удалить (или закомментировать) блок логики первого запуска: проверку `magic-book-initialized` и связанную очистку `entries`.
- Загрузка `entries` из `localStorage` остаётся как есть — пользователь видит сохранённые слова и продолжает добавлять новые.
- Сам ключ `magic-book-initialized` больше не используется (можно оставить в `localStorage` без вреда — читать его не будем).

## Обновить memory
- `mem://tech/session-management` — отразить, что автоматическая очистка при первом запуске отключена; данные сохраняются между сессиями полностью.

## Что НЕ трогаем
- Логику сохранения `entries` в `localStorage` при добавлении слова.
- Валидацию дубликатов, пагинацию, анимации, кнопки, FinalBook, ControlBar.

## Критерии приёмки
- При перезагрузке страницы все ранее внесённые слова остаются в книге.
- Пользователь может продолжать добавлять новые слова к существующему списку.
- Никакой автоматической очистки при загрузке не происходит.
