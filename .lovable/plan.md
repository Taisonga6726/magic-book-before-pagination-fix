

## Цель
При нажатии «📖 читать книгу» открывать книгу с **первой страницы** (с первого внесённого слова), а не с последней.

## Анализ
Сейчас при переключении в `mode === "preview"` / `"reading"` используется текущее значение `currentPage` в `MagicBook`, которое сохраняется между режимами (см. memory `navigation-logic`). Поэтому если пользователь добавил слово на последней странице и нажал «читать книгу» — открывается последняя.

Нужно: при входе в режим чтения принудительно сбрасывать `currentPage` в 0 (или вызывать `setCurrentPage(0)` из родителя через проп / эффект).

## Что проверить (read-only сейчас)
- `src/pages/Index.tsx` — где переключается `mode`, как пробрасывается в `MagicBook`.
- `src/components/MagicBook.tsx` — как управляется `currentPage`, есть ли проп для внешнего сброса.
- `src/components/ControlBar.tsx` — кнопка «📖 читать книгу» вызывает `setMode("preview")`.

## План правки

### Вариант (минимальный, без ломки логики добавления слова):
**Файл: `src/components/MagicBook.tsx`**
- Добавить `useEffect`, который при изменении `mode` на `"preview"` или `"reading"` (mode прокидывается пропом или определяется внутри) сбрасывает `currentPage` в 0.
- Если `mode` сейчас не приходит пропом — добавить новый проп `resetToFirstPageKey` (число/строка), и при изменении этого ключа делать `setCurrentPage(0)`.

**Файл: `src/pages/Index.tsx`**
- В обработчике перехода в `preview` (кнопка «читать книгу») инкрементировать `resetToFirstPageKey` перед `setMode("preview")`. Это даст явный сигнал «открой с начала», но не будет сбрасывать страницу при добавлении новых слов в режиме `form`.

### Что НЕ трогаем
- Логику пагинации, добавления слова (страница остаётся на актуальной при вводе нового слова — memory `navigation-logic`).
- Анимации flip, корешок, кнопки, финальный экран.

## Критерии приёмки
- Нажатие «📖 читать книгу» из любого места всегда открывает книгу со страницы 1 (первое внесённое слово).
- В режиме `form` при добавлении нового слова страница НЕ сбрасывается (поведение сохранено).
- Перелистывание внутри `preview`/`reading` работает как раньше.
