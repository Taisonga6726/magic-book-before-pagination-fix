

# План: Сдвинуть левую страницу левее к орнаменту

## Проблема (по скрину)
Красные стрелки указывают на зазор между левым орнаментом и началом текста. Сейчас `left: 20%` — слишком далеко вправо. Нужно сдвинуть весь блок (включая нумерацию `— 1 —`) левее, чтобы текст и номер страницы выровнялись по внутреннему краю левого орнамента.

## Изменение
`src/components/FinalBook.tsx`, строка 177:

```
left: "20%" → "17%"
width: "26%" → БЕЗ ИЗМЕНЕНИЙ (правый край: 17 + 26 = 43%, отступ от корешка сохраняется)
top, height, padding — БЕЗ ИЗМЕНЕНИЙ
```

Сдвиг −3% влево (~40px на 1311px) → текст и номер страницы прижмутся к внутреннему краю левого орнамента, как просят красные стрелки.

## Что НЕ затрагиваю
- Правую страницу
- Ширину блока (26%) — асимметрия по горизонтали ОК, т.к. левый орнамент визуально смещён
- Вертикальные границы, пагинацию, шрифты, реакции, нумерацию

## Проверка
1. `navigate_to_sandbox` 1311×861.
2. `screenshot` — убедиться, что:
   - левый край текста и номера `— 1 —` касается внутреннего края левого орнамента (без зазора);
   - правый край не наезжает на корешок;
   - вертикальная нумерация записей (1, 2, 3...) идёт ровной линией у орнамента.
