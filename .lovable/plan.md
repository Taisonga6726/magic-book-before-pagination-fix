

# План: динамическая пагинация по высоте вместо фиксированного лимита

## Суть изменения
Убираем `ENTRIES_PER_PAGE = 6`. Вместо этого — реальное измерение высоты контейнера. Если запись не помещается — автоматическое перелистывание на новую страницу.

## Техническая реализация

### MagicBook.tsx

**1. Удалить `ENTRIES_PER_PAGE` и всю логику на его основе** (строки 18, 61, 105-106, 135-140).

**2. Новая структура данных — страницы по высоте:**
- Хранить `pages: Entry[][]` — массив страниц, каждая содержит записи, которые реально поместились.
- При `handleSave`: добавить запись на текущую страницу → через `useEffect` + ref измерить `scrollHeight` vs `clientHeight` контейнера → если переполнение — убрать последнюю запись с текущей страницы, создать новую страницу, перенести туда запись, автоматически перелистнуть.

**3. Механизм измерения переполнения:**
```
const rightPageRef = useRef<HTMLDivElement>(null)

useEffect(() => {
  const el = rightPageRef.current
  if (el && el.scrollHeight > el.clientHeight) {
    // Переполнение — перенести последнюю запись на новую страницу
    // + авто-перелистывание со звуком и анимацией
  }
}, [entries, currentPage])
```

**4. Кнопка «далее →»:**
- Показывать когда: есть следующая страница ИЛИ текущая переполнена.
- При клике на пустую следующую страницу — создать её (`currentPage + 1`).

**5. Кнопка «завершить книгу ✦»:**
- Остаётся видимой всегда (когда есть записи).
- Позиция не меняется.

**6. Проверка дубликатов:**
- В `handleSave` добавить проверку: если `entries.some(e => e.word.toLowerCase() === word.trim().toLowerCase())` — показать «СЛОВО ЕСТЬ УЖЕ В СЛОВАРЕ!» красным, не сохранять.

### FinalBook.tsx

**1. Удалить `ENTRIES_PER_PAGE`.**

**2. Аналогичная логика:** использовать ref для измерения контейнера. Распределять записи по страницам динамически:
- Рендерить записи одну за другой.
- Когда `scrollHeight > clientHeight` — остановиться, остальные на следующую страницу.
- Предварительный расчёт через скрытый контейнер или двухпроходный рендер.
