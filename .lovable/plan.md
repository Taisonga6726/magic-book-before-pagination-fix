

## Диагноз по скриншоту

Вижу на левой странице между записями 20–23 явную «лесенку» (красная линия пользователя): номера `20.`, `21.`, `22.`, `23.` стоят НЕ по одной вертикали — каждый следующий чуть правее предыдущего. Правая страница (24–27) — ровно, всё по одной вертикали.

## Причина

В `src/components/FinalBook.tsx` колонка номера сейчас `minWidth: 1.4em` без фиксированного `width`. Flexbox с `min-width` + `flex-shrink: 0` всё равно позволяет колонке расширяться под содержимое. Когда номер двузначный (`20.`, `21.` …), реальная ширина колонки = ширине текста `20.`, но из-за разной ширины глифов italic-шрифта Cormorant Garamond (`20.` и `23.` визуально разной ширины) точка съезжает, и слово начинается на разной горизонтали → лесенка.

На правой странице её нет, потому что там цифры `24–27` визуально близки по ширине, и эффект незаметен — но проблема та же по природе.

## Решение

В `src/components/FinalBook.tsx`, в `renderEntry` (унифицированная функция для обеих страниц):

1. Заменить `minWidth: "1.4em"` на жёсткий `width: "2.2em"` (фикс под двузначные `99.`) + `flexShrink: 0` оставить.
2. Дополнительно `textAlign: "left"` уже стоит — оставить. Точка `.` тогда всегда на одной горизонтали, и слово справа от неё начинается строго по одной вертикали для всех записей.
3. Синхронизировать `measure`-блок (где считается высота для пагинации) — там тоже поставить `width: 2.2em` вместо `min-width: 1.4em`, чтобы расчёт переноса совпадал.

Никакого `padding-right`, никакого зазора — просто фиксированная колонка под номер. Между точкой и словом — естественный единичный пробел от flex-gap (его нет → один gap нужен: либо через пробел в JSX, либо `gap: "0.3em"` на родителе). Сейчас flex без gap, слово начинается вплотную к границе колонки → нужен либо `gap: 0.25em` на родительском flex, либо чуть-чуть `paddingRight: 0.15em` у колонки номера — минимально, ровно как одиночный пробел.

## Что меняется (точечно)

**Файл:** `src/components/FinalBook.tsx`

- Колонка номера в `renderEntry`: `minWidth: "1.4em"` → `width: "2.2em"`, добавить `paddingRight: "0.15em"` (узкий зазор как одиночный пробел).
- `measure`-блок: те же значения для совпадения с пагинацией.

## Что НЕ трогаю

- Правую страницу как разметку — она уже использует ту же `renderEntry`, изменения применятся одинаково.
- Координаты страниц, `top: 19%`, `height: 60%`, ширины колонок (18%/31%, 54%/26%).
- Шрифты, цвета, ink-эффект, реакции, перелистывание, орнамент.
- `marginBottom: 0.6em` между записями (дыхание).

## Результат

- Все номера (`1.`, `2.`, … `9.`, `10.`, `11.`, … `99.`) выстроены по **одной вертикали** на обеих страницах.
- Все слова начинаются по **одной вертикали** — лесенка исчезает.
- Между номером и словом — естественный одиночный пробел.
- Описание остаётся под словом (hanging indent).
