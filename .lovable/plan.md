

# Plan: Заменить картинку FinalBook на book.png из MagicBook

## Корень проблемы
FinalBook использует **другую картинку** (`/images/open-book.png`), а MagicBook — `@/assets/book.png`. Координаты страниц уже совпадают (`left: 54%, top: 18%, width: 26%, height: 60%`), но орнаменты на разных картинках расположены по-разному. Пока используется другая картинка, подгонка координат — бесконечный цикл.

## Изменения в `src/components/FinalBook.tsx`

### 1. Заменить картинку на ту же, что в MagicBook
```tsx
// Было:
const bookFinalImg = "/images/open-book.png";

// Станет:
import bookFinalImg from "@/assets/book.png";
```

### 2. Координаты — оставить как есть
Правая страница уже идентична MagicBook: `left: "54%", top: "18%", width: "26%", height: "60%"`, `padding: "12px 2px 40px 4px"`.

Левая страница: `left: "20%", top: "18%", width: "26%", height: "60%"`, `padding: "12px 4px 40px 2px"` — зеркало правой.

### 3. Без визуальных эффектов
Никаких glow, анимаций, частиц — чистая стабильная основа.

## Результат
Одна картинка = одни координаты = текст попадает ровно между орнаментами на обеих страницах, как в MagicBook.
